HTML
텍스트 편집기 - 포멧 - 일반텍스트 만들기 - 파일 - 저장 - .html 형식으로 저장


하이퍼 텍스트 속성
<strong> </strong> -> 글씨 강조
<h1> <h1> -> 제목, 뒤의 숫자가 작을수록 글자 크기가 작아짐 
<a href="링크주소" title ="텍스트 설명내용"> </a> -> 링크걸기 및 설명

태그 중첩과 목록
<li> </li> -> 태그, 보통 단독사용X
<ul> </ul> ->리스트를 묶는 태그, 그룹화 용도, 숫자 없이 나열
<ol> </ol> -> 리스트를 묶는 태그, 그룹화 용도, 숫자를 붙여 나열

문서의 구조
<tilte> </title> -> 탭의 제목 설정
<meta charset="utf-8"> -> html 로딩시 글씨 깨질경우 해당 코드 입fur

문서는 head(서문), body(본문),
Head에는 title과 meta 코드를 넣기로 암묵적으로 약속 
<html> </html> -> head, body 태그를 감싸주는 역할


Doctype
ㄴ 어떤 방식으로 작성된 html 코드인지 브라우저에게 알려주는 역할
html버전에 따라 적당한 doctype 선언

태그
<p> </p> -> 단락 구분
<br> -> 줄바꿈, 닫는태그 따로 필요 없음
<img src="파일이름.jpg" width="" height="' alt="이미지 깨졌을때 텍스트" title="이미지 설명내용"> -> 이미지 삽입

표 -> 레이아웃 만들때도 자주 사용
<td> </td> ->표에 들어가는 항목들을 td로 감싸기
<tr> </tr> -> 같은 행에 속하는 td 태그들을 tr로 묶기
<table border ="테두리 두께"> </table> -> 표애 속하는 내용들을 묶기

입력양식
<input type="text" name="해당 객체의 이름" value="기본값"> -> 사용자가 정보를 입력할 수 있는 텍스트 생성
<input type="password" name="해당 객체의 이름"> -> 내용이 안보이는 텍스트
<input type="submit"> -> 제출 버튼 생성
<form action="정보를 보낼 주소"> </form> -> 사용자가 정보를 보낼수 있도록 지정
<textarea cols="입력할 글자수" rows="줄 수"> 기본값 </textarea> -> 여러줄 입력가능

선택
*텍스트창
<option value="~~~"> </option> -> 항목들 선택가능, 선택시 value에 있는 값 전송 
<select name="" multiple> </select> -> 같은 행에 속하는 option 태그들을 묶기, name은 중복 불가, multiple은 ctrl 누를시 다중선택 활성화 
<form action="http://~~~"> -> 해당 내용을 주소로 전송

*라디오 버튼(한개의 버튼만 선택 가능)
<input type="radio" name="" value="" checked> -> 라디오 버튼, 같은걸로 묶고 싶을시엔 name 같은걸로 묶기 checked는 페이지 열었을때 기본으로 선택되는 것


*체크박스(다중 선택가능)
<input type="checkbox" name="" value="" checked> -> 체크박스 생성


버튼
<input type="submit" value=""> -> 제출버튼
<input type="button" value=""> -> 일반 버튼
<input type="reset"> -> 정보 초기하

데이터 전송
<input type="hidden" name="" value=""> -> 눈에 보이지 않는 정보 전송

컨트롤의 제목
<label> </label> -> 텍스트의 제목, 텍스트끼리 묶는것 가능
ex)<label for="id_txt"> ~~~ </label>
<input id="id_txt" type="text" name="id" value= "default value">  -> 위의 ~~~ 텍스트 클릭시 text라는 텍스트창으로 커서 이동
다른 방식으로는 <label> </label>로 전체를 감싸는 방식이 있음 

Method
url에 정보 노출하지 않고 정보 전달시 -> post
<form action="~~~~" method="get"> -> method지정하지 않을시 기본적으로는 get방식(url에 정보 노출)
<form action="~~~~" method="post"> -> url에 정보노출x
ㄴ서버엔지니어의 권한, 지정부탁

파일 업로드
<form action="~~~~" method="post" enctype="multipart/form-data"> -> method와 enctype는 저 형식으로 필수
<input type="file" name="~~~">

meta태그
<meta charset="utf-8"> -> utf-8형식으로 파일 저장
<meta name="description" content="~~~"> -> html파일 요약자료
<meta name="keywords" content="~~~"> -> html파일 키워드 요약
<meta name="author" content="~~~"> -> 저자 표시
<meta http-equiv="refresh" content="30"> -> 30초 간격으로 refresh

의미론적 태그
<header> </header> -> 화면의 상단에 위치하는 사이트나 문서의 전체적인 정보를 정의
<footer> </footer> -> 화면의 하단에 위치하는 사이트나 문서의 전체적인 정보를 정의
<nav> </nav> -> 문서의 네비게이션 항목을 정의
<article> </article> -> 본문
<section> </section> -> 문서의 구획들을 정의
참고: https://opentutorials.org/course/2039/10954
  

모바일 지원
우클릭 - 검사 - 스마트폰 아이콘 클릭 - 기기선택
<meta name="viewport" content="width=device-width, initial-scale=1.0"> -> 웹사이트를 모바일에서 볼때 맞추도록 하는 코드

외부문서삽입

<iframe src="~~~~~" width="" height="", sandbox> -> sandbox는 위험요소 원천차단

<video> <source src="videos/small.ogv"> </video>
